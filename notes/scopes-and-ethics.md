# Scope and Ethics

## Scope
This project was conducted on systems that I own or that I had explicit permission to test.  
The purpose of the work was to practice network reconnaissance, service discovery, and security documentation in a controlled environment.

The assessment was limited to passive and non-intrusive reconnaissance activities, including:
- Host discovery.
- Port scanning.
- Service and version detection.
- OS fingerprinting.
- Script-based enumeration using safe default checks.

No exploitation, persistence, credential theft, denial of service, or unauthorized access attempts were performed.  
The goal was to understand exposed services and document the results for educational and portfolio use only.

## Ethics
I followed responsible security practices throughout the project.  
All testing was performed with clear authorization, and only the minimum necessary scanning was used to gather information.

I did not:
- Scan public targets without permission.
- Attempt to bypass authentication.
- Run destructive or disruptive tests.
- Modify target systems.
- Collect or expose sensitive data beyond what was required for analysis.

Any findings were recorded accurately and presented in a professional manner.  
If a service appeared risky or unnecessary, I noted it as a hardening concern rather than a vulnerability to be exploited.

## Responsible Use
Reconnaissance tools can reveal valuable information about a system, so they should only be used within a legal and ethical scope.  
In real environments, all scanning should be approved by the system owner, security team, or lab rules before execution.

## Statement
This project is intended for learning, portfolio development, and defensive security practice.  
It reflects a security-minded approach focused on understanding exposure, improving documentation, and supporting safer system configurations.
